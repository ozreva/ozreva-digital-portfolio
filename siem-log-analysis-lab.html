SIEM and Log Analysis Lab

By ozreva | August 25, 2025

Introduction

Effective log management and event correlation are essential for detecting attacks, investigating incidents, and responding to threats in enterprise environments. To gain hands-on experience with these capabilities, I designed and implemented a SIEM and Log Analysis Lab built around Splunk Enterprise. This project emphasized log aggregation, normalization, correlation, and alerting across multiple operating systems and security devices.

The lab showcased how centralized log analysis can be leveraged for faster detection and improved visibility of critical security events.

SIEM Deployment and Log Aggregation

The cornerstone of the lab was a deployment of Splunk Enterprise with Universal Forwarders installed on various endpoints and systems:

  • Windows systems forwarding Security, System, and Application Event Logs (including Sysmon).
  • Linux systems sending authentication and system logs.
  • pfSense firewall exporting traffic and security events.
  • Suricata IDS alerts for intrusion detection.

This setup ensured that all logs from servers, endpoints, and network devices were centrally collected into Splunk for unified analysis.

Why this was done: Distributed logs without a centralized collection point complicate detection and make incident response slow and incomplete. Aggregation in Splunk simplified visibility and ensured no events were missed.

Log Normalization and Correlation

To enable meaningful analysis, I normalized events from heterogeneous sources:

  • Windows Event IDs and Sysmon telemetry for process execution and authentication trends.
  • Linux auth logs for privileged access and failed login attempts.
  • Suricata alerts for intrusion activity at the network layer.

Splunk’s ability to parse, normalize, and correlate events from different environments allowed me to build a unified security picture—connecting endpoint activity with network IDS alerts for comprehensive incident visibility.

Detection Dashboards and Correlation Rules

Inside Splunk, I designed custom dashboards for real-time monitoring and historical exploration of security events. Key correlation rules were developed to highlight:

  • Brute force attempts across SSH and RDP sessions.
  • Privilege escalation events detected in system logs.
  • Unusual network activity correlated with Suricata IDS alerts.

These dashboards provided an immediate view of the environment’s security posture while rules automated much of the detection workflow.

Attack Simulation and Alert Validation

To validate the effectiveness of the detections, I used a Kali Linux VM to launch simulated attacks, including:

  • SSH brute force attempts against Linux servers.
  • RDP brute force attacks against Windows endpoints.

These attempts were logged by the target systems, forwarded to Splunk, and correlated with Suricata alerts, triggering the expected detections in real time. This confirmed that the dashboards and detection rules were correctly tuned to identify hostile behavior.

Real-Time Alerting and Incident Response

Finally, I configured real-time alerts within Splunk for high-severity events such as repeated failed logins, privilege escalation, or simultaneous alerts across multiple systems. Alerts were delivered as notifications, ensuring timely response to simulated incidents.

Why this was done: Detecting and logging suspicious behavior is important, but timely response is critical. Real-time alerts shorten the gap between detection and action, providing a significant improvement to incident response workflows.

Conclusion

The SIEM and Log Analysis Lab demonstrated how Splunk can be deployed in an enterprise-like environment to deliver deep visibility into security events across systems, endpoints, and networks. By aggregating, normalizing, and correlating logs, the lab mirrored best practices for real-world detection and response.

Key outcomes included:

  • Centralized log aggregation across Windows, Linux, firewalls, and IDS sensors.
  • Consistent log normalization for improved correlation.
  • Actionable detection dashboards and correlation rules for brute force, privilege abuse, and anomalous activity.
  • Validation of defenses through attack simulations.
  • Improved incident response with real-time alerting.

The result was a fully integrated SIEM lab where detection, monitoring, and alerting worked together to improve visibility and readiness—skills directly applicable to enterprise security operations.
